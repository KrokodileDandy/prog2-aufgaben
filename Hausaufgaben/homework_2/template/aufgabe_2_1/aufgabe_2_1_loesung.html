Java I/O-Streams

Unterschied zwischen buffered und unbuffered Lesen und Schreiben:

Alle Lese- und Schreib-Anfragen von unbuffered Streams werden vom zugrundeliegenden Betriebssystem bearbeitet (sehr ineffizient). Buffered Input-Streams lesen Daten aus einem Speicherbereich (Buffer); das native Input-API wird nur aufgerufen, wenn der Buffer leerläuft. Buffered Output-Streams schreiben Daten in einen Buffer und das native API wird nur aufgerufen, wenn der Buffer vollläuft.

Einlesen und Schreiben von Text-Dateien:

Ich würde die Buffered Streams zum Lesen und Schreiben von Text-Dateien verwenden, weil hier keine Serialisierung von Nöten ist (sondern nur „reiner“ Text) und man am wenigsten Zwischenschritte benötigt, um auf eine Datei zuzugreifen.

Data Streams würde ich verwenden, wenn ich komplexe Daten habe, welche ich nicht erst per Hand in einem bestimmten Text-Format formatieren möchte.
Für Object Streams gilt das gleiche, weil sich diese für das Speichern von ganzen Objekten eignen, ohne dass ich mich darum kümmern muss, z.B. wie in einem CSV-File für jedes Attribut eine Spalte zu reservieren.

Übersicht Java-Streams

BezeichnungFunktion
Byte Stream Bearbeiten I/O von rohen Binärdaten. Alle Byte-Stream Klassen erben von InputStream und OutputStream.
Character Stream Bearbeiten I/O von Characters. Außerdem bearbeiten sie automatisch die Übertragung zu und von dem lokalen Character-Set.
Buffered Stream Optimieren Input und Output, indem sie die Anzahl der Aufrufe der nativen API reduzieren.
Data Stream Bearbeiten binäre I/O-Repräsentation von primitiven Datentypen und String-Werten.
Object Stream Bearbeiten binäre I/O-Repräsentationen von Objekten.
